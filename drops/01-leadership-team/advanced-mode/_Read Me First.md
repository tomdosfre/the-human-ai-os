# Advanced mode — read this first (not something you upload)

This file is for you, the human, to read. It doesn't go into Claude. The
`team/` folder next to it is what you actually use.

## What advanced mode is

The same six leadership-team seats as the starter drop — Chief of Staff,
CFO, CPTO, CISO, CGO, CMO — rebuilt for someone who's comfortable driving
Claude directly and wants less ceremony, more depth. Concretely, versus the
starter drop:

- **No Onboarding Buddy.** The starter files run a one-minute guided setup
  the first time you paste them (a greeting, two skippable questions, a
  naming step). These files skip straight to substance — mandate, reasoning,
  work.
- **Deeper reasoning.** Each seat's blend is folded in with the full routing
  logic (which voice leads which kind of call), a standing decision
  sequence for that domain, and a self-check the seat runs on itself before
  asserting something as current fact.
- **Seats aware of each other.** A shared `_team-protocol.md` file gives
  every seat the same roster, the same handoff rules, and the same hard
  gate — nothing irreversible (money, contracts, credentials) is ever
  decided by the system alone, always proposed back to you.
- **A deeper authoring standard.** `seat-standard.md` is the full pattern
  for building your own seat, not the lite version — if you want a seat for
  something the six don't cover, this is the spec to build against.

## How to install it

**Best: upload the whole `team/` folder to a Claude Project.** That's what
makes the seats aware of each other — each one can read `_team-protocol.md`
and knows the full roster, not just itself. Once it's in a Project's
knowledge, open a new chat in that Project and just start talking to any
seat by name, or ask "who should I talk to about X."

**Also works: paste one seat file on its own.** Each file in `team/` is
fully self-contained — you can paste just `cfo.md` into a fresh chat and get
a working CFO with no dependency on the rest. You lose the cross-seat
awareness (it won't know to hand you to another seat), but the seat itself
works exactly the same.

Either way, there's no setup step. Paste it (or open the Project) and start
with whatever's actually on your mind.

## How to extend it

If none of the six seats cover something you need — a side project, a
rental property, a team you manage — read `team/seat-standard.md`. It's the
full authoring pattern: mandate, blend, tie-breaker, what it pushes back on,
and the reporting shape, with a worked example at the end. Ask any seat to
draft the new one for you in that shape, or write it yourself.
